Прописываем маршруты для pip, stm и т.п. по определенному шлюзу:

https://jtprog.ru/windows-route-add/

Все команды от имени администратора:

Подготовительная часть - приводим в состояние по умолчанию:
Требуется, если выполнялись какие-то действия и они не привели к результатам

1. Удалить все основные маршруты (см. по списку в route print)
	route delete 0.0.0.0
	...

2. Добавить основной маршрут
	route -p add 0.0.0.0 mask 0.0.0.0 172.16.6.56

Основная часть:

3. В свойствах адаптера указать шлюз:172.16.6.100 с автоматической метрикой, проверить:
	route print
	Локалка должна работать + локальные мессенджеры

4. Прописываем маршрут для pip
	route -p add 151.101.64.223 mask 255.255.255.255 172.16.6.100
	route -p add 151.101.13.63 mask 255.255.255.255 172.16.6.100

	Проверить трассировку: В первой строке должно быть 172.16.6.100
	tracert 151.101.64.223
	tracert 151.101.13.63
	
	Проверка от обычного пользователя:
	pip cache purge
	pip uninstall opencv-python
	pip install opencv-python
